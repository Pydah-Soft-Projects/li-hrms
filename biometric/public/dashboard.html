Sync attendance to MongoDB (TCP)

Reads punches from the terminal (full buffer), then inserts into this server’s database. Leave both dates empty for normal incremental sync (only new punches after last sync). With a range, only punches on those calendar days (server local time, same as backup filters) are inserted—useful for backfill.

What each action does — expand for details

TCP backup — Pull slim rows via port 4370 (getAttendances). Filters below apply to preview / TCP backup / clear validation.

Fresh ADMS backup — Queues DATA QUERY ATTLOG, waits for new device POSTs, saves JSON with full push bodies (can take ~2 min).

Export stored (DB) — Writes ATTLOG batches already in MongoDB; optional dates filter when each batch was receivedAt.

Clear device — Always runs Fresh ADMS file first (queue + wait, up to ~2 min), then clears punch logs on the machine. Optional extra TCP snapshot during clear if checked below.
